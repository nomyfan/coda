# Code review

When reviewing a change:

1. Read the diff in full before commenting.
2. Flag correctness bugs first, then readability, then missing test coverage.
3. Prefer concrete suggestions ("extract X into Y") over vague notes.
4. Keep the summary to the three or four highest-impact findings.
