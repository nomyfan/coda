# Docs workspace instructions

This file is the **workspace knowledge** for any agent rooted here (the
`docs-writer`). It is separate from the root workspace's `AGENTS.md` — that is
the whole point of a per-agent workspace.

## Style

- Use sentence-case headings.
- Keep paragraphs short; prefer lists for steps.
- Every new doc starts with a one-line summary of what it covers.
